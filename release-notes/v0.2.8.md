# 0.2.8 (2026-04-20)

這個 patch release 主要收兩件事：固定 Android release signing，以及把 QuickJS 啟用後在 CI 暴露出的 JS/CSS 回歸修乾淨。從這一版開始，release APK 會使用固定的正式 keystore 簽名，後續版本可維持一致的升級路徑。

## 亮點

- Android release APK 改用固定 keystore 簽名
- GitHub Actions release workflow 補齊正式 signing secrets 流程
- 修掉 6 個 QuickJS / JS / CSS 相關 CI 回歸

## 詳細更新

### Android 發版

- `android/app/build.gradle.kts` 現在支援從 `android/key.properties` 或 CI 環境變數讀取正式 release signing
- release workflow 會從 GitHub Secrets 還原 keystore，並用固定 key 簽名 APK
- 這是第一個走正式 release key 的版本；若裝置上仍是舊的 debug-signed 安裝包，升到這一版可能需要最後一次手動卸載重裝

### CI / 測試回歸

- 修正 3 個 integration tests 的 Flutter binding 初始化問題
- 修正 CSS selector compat，避免 `^=` 等 attribute selector 被誤處理
- 修正 nested async JS rewrite，讓巢狀 `java.ajax(...)` 正確傳遞內層結果
- 修正 Promise bridge 測試上下文，避免未定義全域變數導致假失敗
