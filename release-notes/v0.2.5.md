# 0.2.5 (2026-04-20)

這一版的重點不是再擴大書源掃描，而是把已經驗過的結果更完整地收進產品本體，並把 release 流程、文件與 CI 收口。

## 亮點

- 書源管理頁朝 Legado 的操作流全面收斂，主頁入口與單列動作更直接
- 修正從搜尋結果進入詳情後，「開始閱讀」直接變成「暫無章節」的問題
- 補上單書源搜尋入口，從書源管理即可直接在指定來源內搜尋
- QuickJS 依賴測試改成 CI-safe，缺少 native library 的 runner 不再誤報失敗

## 詳細更新

### 書源管理

- 主頁收斂為 `排序 / 分組 / 更多` 三個入口
- 列表改為平鋪式管理介面，域名分組時直接顯示 host header
- 單列補齊 `編輯 / 更多 / 啟用開關 / 發現狀態`
- 批量工具列新增 `啟用發現 / 禁用發現 / 連續選取`
- 單源更多選單新增 `在此書源中搜尋 / 登入書源 / 啟停發現`

### 閱讀鏈路

- 詳情頁已抓到的章節清單現在會直接交給閱讀器
- 閱讀器啟動時會優先使用這批初始章節，不再被空的本地 DAO 結果覆蓋

### 測試與文件

- 修正 CSS self-match、attribute selector 與章節標題重複相關回歸
- QuickJS 依賴測試在無 native library 的環境下自動 skip
- 更新 README、roadmap、database、reader architecture 與下一階段交接文檔
