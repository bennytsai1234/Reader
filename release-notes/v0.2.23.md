# 0.2.23 (2026-04-24)

這個版本把閱讀器 scroll/slide 開書穩定性與網路書快取策略正式收進 release。重點是讓網路書更接近本地書的開啟體驗，並避免閱讀畫面在內容已抓回後仍停在空白狀態。

## 亮點

- 網路書改為 cache-first 內容載入，書架內網路書會長期保存章節原文
- 非書架網路書新增 transient DB cache，避免每次開啟都完全重新抓取
- 加入書架或排入離線快取時，會把 transient cache 提升成正式章節內容
- scroll restore 未確認前不再覆寫進度，降低重開後跳到章首的風險
- scroll restore 期間阻擋無效滑動，等畫面真正定位後再恢復互動
- scroll 與 slide 模式若遇到空白 viewport，會自動補載/重新呈現目前章節
- 內容已抓回但分頁被 viewport/inset 變動打斷時，會用最新 layout 補分頁

## 詳細更新

### 網路書快取

- 新增 `ReaderTempChapterCaches` Drift table 與 DAO，資料庫 schema 升到 v10
- 新增 `ReaderChapterContentCacheRepository`，統一判斷書架長期快取與非書架暫存快取
- 書架內網路書內容寫入正式 `chapters.content`
- 非書架網路書內容寫入 transient cache，不污染正式章節資料
- 書籍加入書架或建立離線快取任務時，會嘗試提升既有 transient cache
- 書架內網路書提高鄰近章節預快取範圍與預載並行度

### 閱讀定位與空白畫面

- restore 尚未完成時不再把暫態 scroll offset 寫成閱讀進度
- scroll restore 改由可見 anchor confirmation 完成，不再只靠跳轉 callback 提前釋放
- restore 使用同一條 anchor padding 語義，降低定位與確認基準不一致
- 內容頁、載入頁與背景 placeholder 都保留點擊喚出選單能力
- 畫面 ready 但沒有可見文字時，下一個 frame 會自動拉起目前章節載入
- raw content 已存在但分頁為空時，會直接以最新 pagination config 重新分頁，不重新打網路
