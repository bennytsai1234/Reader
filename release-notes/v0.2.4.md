# 0.2.4 (2026-04-20)

這一版把前一階段的書源相容性與產品策略正式收斂進 App，本次重點不是再擴功能，而是把「可維護的 Flutter 小說閱讀器」這條主線補完整。

## 亮點

- 書源相容層大幅補強，對齊大量 Legado 規則語義與解析行為
- 書源校驗結果正式接入 App，狀態、標籤與清理建議可直接在來源管理頁查看
- 閱讀失敗時可直接自動換源或手動換源，並保留閱讀進度
- 移除以 `ms` 響應時間為中心的 UI 與排序，改以實際可用性為核心
- 純小說產品策略落地，非小說源、下載站、登入牆來源可明確隔離或清理

## 詳細更新

### 書源引擎與相容層

- 補齊多個 URL / Regex / CSS / JsonPath / JS bridge parity 缺口
- 改善 `POST redirect`、JSONP、async JS 與舊式 Regex 書源兼容
- 修正搜尋、詳情、目錄、正文之間的資料傳遞與章節解析穩定性

### 書源管理與校驗

- 新增書源狀態體系與執行期策略
- 校驗後可直接查看結果摘要，並批次清理建議刪除來源
- 搜尋失效源會自動排除出預設搜尋池
- 需要登入、下載站、非小說源可集中標記與處理

### 閱讀器

- 正文或章節失敗時，閱讀器內直接提供恢復入口
- 支援自動找同名同作者可用源並切換
- 手動換源會保留目前閱讀進度
