# 0.2.25 (2026-04-25)

這個版本集中修正網路書源在閱讀器開書、滾動模式退出重進、平移模式 restore 時的非同步定位 race。正式閱讀位置收斂到 `chapterIndex + charOffset`，只有經過真實 viewport anchor 確認的位置才會持久化，降低空白畫面、回章首與平移模式錯頁跳回的風險。

## 亮點

- scroll mode 不再把 placeholder 或未確認 viewport 寫成正式進度
- 退出或暫停時，scroll 進度 flush 會拒絕 unconfirmed anchor 與未解析章節內容
- slide restore 會先確認目標頁存在，解析失敗不再 fallback 到全域第 0 頁
- PageController reset 已在目標頁時會清除 stale navigation token
- pinned slide target 到達後即使沒有 `onPageChanged` 也會清除
- 首次 visible data 出現後由 app 主動補兩幀 repaint，避免依賴使用者點擊或系統 UI redraw
- 文字 painter 會清背景，並在樣式、padding、掃描線色彩等繪製參數變更時正確 repaint

## 詳細更新

### 閱讀進度

- `ReaderProgressCoordinator.updateVisibleChapterPosition` 新增 `allowProgressCommit`
- `ReadBookController.handleVisibleScrollState` 只有在 `isAnchorConfirmed=true` 時允許 progress commit
- `_persistCurrentScrollLocationNow` 加入 `visibleConfirmed`、最後一次 scroll anchor confirmation 與章節內容可解析檢查
- 未確認 scroll viewport 仍可更新 UI 暫態與預載，但不會更新 committed/durable progress

### 平移模式 restore

- 新增 `ReaderRuntimeController.globalSlidePageIndexForLocation`
- 初始 slide restore 改為 pin target、refresh slide pages、驗證 snapshot/location 後才 reset controller
- 解析不到目標頁時保持原頁，不回退到 page 0
- 新增 pinned target 到達後的顯式清除流程
- `SlidePageController.jumpTo` 新增 already-at-target callback，避免 pending restore/chapterChange token 殘留到下一次翻頁

### 首次繪製

- `ReadViewRuntime` 首次 visible data refresh 改為 generation-based，並補第二幀 repaint
- `_TextPagePainter` 非 auto paging 分支傳入背景色
- `_TextPagePainter.paint` 先清背景再繪文字
- `shouldRepaint` 補齊 style、padding、scan line 與背景等繪製依賴
