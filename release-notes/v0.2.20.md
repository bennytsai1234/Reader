# 0.2.20 (2026-04-23)

這個版本集中完成 Reader 最近一輪 TTS 與閱讀定位核心的重構。重點不是只補朗讀 UI，而是把 Reader 的朗讀流程、位置語義、restore 路徑與章節保留策略收斂成可以長期擴充的主幹，並正式把 HTTP TTS 接進 app。

## 亮點

- Reader TTS 從整章直接 `speak`，改成更穩定的 segment-based queue 流程
- 朗讀高亮、scroll follow、restore 與 viewport command 改成共用同一套位置語義
- 正式支援在 app 內切換 `系統 TTS` 與 `HTTP TTS`
- HTTP TTS 已可經由 `AnalyzeUrl` 取得音訊並透過播放器實際朗讀

## 詳細更新

### Reader TTS 核心

- 修正 `charOffset -> localOffset` 命中邏輯，避免落在當前行的 offset 被錯算到下一行
- `startFromOffset()` 成為主要啟動 API，`startFromLine()` 降為相容 wrapper
- 新增 `ReaderTtsPosition`，收斂原本散落的 `ttsStart / ttsEnd / ttsWordStart / ttsWordEnd / ttsChapterIndex`
- `ReadAloudController` 改成 segment queue，pause/resume 以章內 offset 重建，不再依賴 substring 補償
- `ReaderTtsEngine` / `SystemTtsEngine` / `SwitchableReaderTtsEngine` 接上，controller 不再直接綁 `TTSService`

### 閱讀定位與 restore

- `ReaderAnchor` 正式進入 open / restore / viewport command 流程
- scroll mode 的 `restore / visible confirmation / placeholder re-anchor / active navigation / TTS follow` 收斂成同一套 settle state
- pending restore、active navigation target、TTS current/next 章節納入 retention，避免跨章 handoff 或 restore 期間被過早 evict
- local scroll preload 補強，開書與快速滑動時的鄰章準備更積極

### HTTP TTS

- 新增 `HttpTtsEngine`
- 新增 `ReaderTtsEngineFactory`，依設定或書內 override 決定使用 system/http 路線
- 新增 `ReaderTtsSourcePreference`，把 `system` 與 `httpTts:<id>` 正規化成同一套 source key
- 透過 `AnalyzeUrl.create(..., speakText:, speakSpeed:)` 產生 HTTP TTS 請求
- 加入 `just_audio` 播放 adapter，HTTP 回傳音訊可直接在 app 內播放
- 設定頁與閱讀內 TTS 面板都可直接切換朗讀來源

### 資料與設定

- `HttpTtsDao` 補上 `getById()` / `create()`
- `HttpTtsProvider.upsert()` 修正新建 engine 的儲存路徑
- 新增 `PreferKey.ttsSource`
- `SettingsProvider` 與 reader runtime 都會保存並套用目前的 TTS source
