# 0.2.18 (2026-04-22)

這個版本把閱讀器核心收斂工作正式帶進 release，重點是讓閱讀位置、章節切換、restore 與排版重算回到同一套語義，同時把 CI 測試流程收成 plain `flutter test` 就能直接覆蓋 QuickJS 測試。

## 亮點

- 閱讀器核心位置模型收斂到單一 `ReaderLocation` 語義，章節切換與 restore 更穩定
- scroll / slide 的 restore、repaginate、preload eviction 與 progress flush 對齊同一條核心流程
- 預設測試流程不再依賴 QuickJS wrapper script，GitHub Actions 也直接跑 plain `flutter test`

## 詳細更新

### 閱讀器核心

- `ReaderSessionState` 改成以 `committedLocation`、`visibleLocation`、`durableLocation`、`visibleConfirmed`、`generation` 為真源
- `ReaderNavigationController` 收斂成唯一 command pipeline，章節 jump、restore、slider、drawer 選章改走同一條交易流程
- `ReaderChapter` 與 `ReaderChapterMetrics` 成為統一 layout 單元，char/page/local offset 映射集中管理
- repaginate 改成 staged commit，舊 layout 會保留到新 layout ready，再以同一個 `ReaderLocation` 原子替換
- scroll restore completion、persist flush、retained preload set 都補上正式狀態與回歸測試

### 測試與 CI

- `test/test_helper.dart` 會自動從 pub-cache preload QuickJS 動態庫，plain `flutter test` 可直接跑 engine / JS 測試
- `Flutter CI` workflow 改成直接執行 `flutter analyze` 與 `flutter test --reporter compact`
- GitHub Actions checkout 升級到 `actions/checkout@v5`，避開既有 Node 20 deprecation 警告

## 驗證

- `flutter analyze`
- `flutter test --reporter compact`
- GitHub Actions `Flutter CI`
