# 0.2.17 (2026-04-22)

這個版本集中把「基本閱讀器」最近一輪重構收進正式 release。重點不是外觀翻新，而是把閱讀設定、章節切換、閱讀頁殼層與還原流程整理成更穩定的結構，並補上整頁交互回歸測試，降低之後繼續迭代 reader 時的回歸風險。

## 亮點

- 閱讀器設定改成以單一 reader prefs 為真源，減少設定頁和閱讀頁不同步
- 章節切換補上明確 pending flow，drawer、slider、上一章/下一章行為更一致
- 閱讀頁 shell、tap action、exit flow 拆成獨立層，後續改 UI 不再一直撞主 controller
- page-level 測試擴到真實內容點擊、bottom slider、top back、drawer 選章整條路徑

## 詳細更新

### 閱讀設定與排版

- `ReaderPrefsRepository` 成為閱讀器偏好的集中來源
- 閱讀設定頁與閱讀頁內設定改為共用同一套 reader prefs
- 補齊 `letterSpacing`、`textFullJustify`、`showReadTitleAddition`、`readBarStyleFollowPage`、`selectText`、`showAddToShelfAlert` 的保存與還原
- 打點區設定改為走統一的 `ReaderTapAction` 模型

### 閱讀頁與章節導航

- `ReadBookController` 進一步拆出 chapter navigation、shell interaction、auxiliary flow
- `ReaderPage` 改成以 action dispatcher、exit coordinator、shell widget 組裝閱讀頁
- `scroll` 模式翻頁改為 viewport step，而不是直接把翻頁退化成切章
- 章節切換補上 `pendingChapterNavigation` 流程，避免切章重入
- bottom chapter slider、chapter drawer、上一章/下一章按鈕對齊同一條章節導航語義
- chapter drawer 會跟著目前章節與 pending chapter 自動定位

### 交互與穩定性

- `selectText` 開啟時仍保留 tap action wiring，不再互相踩掉
- top/bottom menu 可跟隨頁面主題配色
- 閱讀頁 shell 內容區改成明確占滿 viewport，避免殼層與內容層約束不一致
- exit / add-to-bookshelf 流程獨立為 coordinator，top back 路徑更容易驗證

### 測試與文檔

- 新增 reader page / shell / drawer / exit flow 的 page-level coverage
- 補上 bottom slider 切章、drawer 開目錄選章、top back prompt 加書架等整條 UI 路徑測試
- 更新 reader runtime / decomposition / release 文檔，讓目前結構與發版流程可追
