# 0.2.29 (2026-04-26)

這個版本重整閱讀器底層模型，將閱讀位置統一到 `chapterIndex + charOffset`，並把正文取得流程收斂成 storage-first。Reader 只負責顯示與排版，章節正文補全改由獨立 scheduler 依照目前閱讀章節向後、向前補齊。

## 閱讀器

- 新增 line-first 底層模型，scroll 與翻頁都以 `TextLine.chapterPosition` 作為可保存座標
- scroll mode 以 `LineItem` 作為主要顯示單位，保存與還原直接對齊章節內 char offset
- 翻頁 mode 以 `LineItem` 組成 `PageGroup`，page 只是渲染結果，不再作為 durable 進度
- 移除 manga/image reader 殘留 UI 與舊多層座標轉換，閱讀器聚焦純文字渲染

## 正文儲存

- 新增 `ReaderChapterContents` storage 與 `ReaderChapterContentStore`
- 移除舊 reader temp chapter cache，網路書與本地書都走同一套 storage 模型
- 新增 `ReaderChapterContentStorage` facade，Reader 讀正文時先讀 storage，缺章再 materialize 並寫回 storage
- 封面改走 book cover storage，書籍刪除時可一併清理 Book、Chapters、ReaderChapterContents 與封面資產

## 背景補全

- 新增 `ChapterContentScheduler`，補全順序為目前章、下一章、上一章，然後往後補到結尾，再回頭補前面
- Reader 不再持有全書 `_backgroundContentQueue`，只保留畫面附近排版用的 `_preloadQueue`
- scheduler 不會因為滑動互動暫停；可見章節改變時只重排尚未開始的任務
- Download executor 改用 storage facade，讓下載、儲存、閱讀使用同一條資料通道

## UI

- 移除書籍詳情頁的「背景下載」入口
- 移除設定中的「存儲與下載」頁面
- 更新相關測試，避免舊 cache / 離線快取語意繼續回流
