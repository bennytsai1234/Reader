# 0.2.6 (2026-04-20)

這一版把最近一輪探索頁、閱讀器跟讀體驗與 QuickJS 測試環境的收尾正式打進 release。重點不是再擴大書源審計，而是把「日常使用會碰到的發現與閱讀細節」補到更接近 Legado，並讓本地/CI 測試環境真正可用。

## 亮點

- 發現頁 UI 與操作流進一步對齊 Legado
- 發現規則依賴 JS 時，現在有持久化快取、錯誤回顯與快取回退
- 修正部分書源從詳情進閱讀時的目錄順序與起始章節問題
- 補齊 Linux / WSL / CI 的 QuickJS 測試入口，不再靠手動設環境變數

## 詳細更新

### 發現頁

- 將發現頁收斂為更接近 Legado 的版面：頂部搜尋列、緊湊書源標題列、框線分類區與精簡長按選單
- 長按書源時直接提供 `編輯 / 置頂 / 登入 / 搜索 / 刷新 / 刪除`
- 發現規則若 JS 執行失敗，改為顯式顯示 `ERROR:*` 分類，避免靜默空白

### 發現規則與 JS

- `exploreUrl` 的 JS 結果現在會做持久化快取，重新整理分類會同步清掉記憶體與磁碟快取
- 若 JS 發現規則臨時失敗，但之前已有成功結果，會優先回退到快取分類
- 修正 QuickJS 測試探測邏輯，避免測試本身在 test zone 外誤啟動 runtime

### 閱讀器 / TTS

- 修正部分書源目錄顯示順序與 Legado 不一致，避免「開始閱讀」直接跳到最新章
- 收斂 TTS 跟讀、閱讀頁安全邊界、控制層誤觸翻頁等細節
- 滾動 / 平移兩種閱讀模式的顯示與跟讀同步更穩定

### 測試與工具

- 新增 `tool/with_quickjs_env.sh`
- 新增 `tool/flutter_test_with_quickjs.sh`
- `tool/run_source_validation.sh` 改走同一套 QuickJS wrapper
- GitHub Actions `dart.yml` 改用 wrapper 跑測試
