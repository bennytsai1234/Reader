# Reader x Legado 驗證工作流程

## 目的

- 用於檢查、驗證、review、重現或評估風險，且不立即修改程式碼的任務。
- 除非使用者明確要求修復或實作，驗證流程不直接改 code。
- 一般驗證不要重新執行 Codebase Atlas；使用本工作流程與模組文檔，若發現 Atlas 過時，把它列為驗證發現。

## 工作流程

1. 保留驗證問題、預期行為、風險或檢查範圍。
2. 打開 [reader_legado_index.md](reader_legado_index.md)，選受影響模組或邊界。
3. 閱讀相關模組文檔的目前狀態、上下游、變更入口與已知風險。
4. 選最小有用驗證方法：測試、lint、analyze、build、靜態檢查、smoke、重現步驟或人工 review。
5. 只執行符合所需信心程度的檢查；若要擴大範圍，說明原因。
6. 分別記錄 pass、fail、blocked、skipped、inconclusive。
7. 若需要修復，建議切到 bug、優化或重構工作流程，不要在驗證流程中默默改 code。
8. 若驗證造成文件更新，依交付政策完成；若沒有檔案變更，摘要驗證狀態並說明不需要 commit。

## 驗證摘要

結束時摘要：

- 驗證目標
- 受影響模組或邊界
- 已執行檢查
- 結果
- 阻塞或略過的檢查
- 建議下一步

## 交付政策

提交並推送。若有檔案變更，完成驗證後建立聚焦 commit，確認目標 remote/branch 後推送；若 remote 或 branch 不清楚，推送前必須先詢問。若沒有檔案變更，摘要結果並說明不需要 commit。
