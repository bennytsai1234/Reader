# Reader x Legado 重構工作流程

## 目的

- 用於結構、責任、API 形狀、命名、重複邏輯或模組邊界調整，預期行為應維持不變。
- 不用重構包裝 unrelated feature 或產品行為變更。
- 一般重構不要重新執行 Codebase Atlas；使用本工作流程與模組文檔，只有模組邊界真的改變時更新 Atlas。

## 工作流程

1. 保留使用者原始要求，明確寫下必須維持不變的行為。
2. 打開 [reader_legado_index.md](reader_legado_index.md)，選主模組與必要邊界模組。
3. 閱讀相關模組文檔的目前狀態、上下游、變更入口與已知風險。
4. 依需要查看 `legado` 對應區域，只借鏡責任切分與流程穩定化，不導入新功能。
5. 定義最小重構邊界與明確非目標。
6. 若重構跨模組，依被釐清或移動的責任選主模組；其他模組只做必要介面調整，並分開說明理由。
7. 先找能保護不變行為的測試、分析、build 或 smoke checks。
8. 以聚焦步驟重構，除非這次目標就是改 API，否則保持 public interface 穩定。
9. 驗證行為與 downstream integration 沒有回歸。
10. 若結構、責任、API 或流程改變，更新受影響的 Atlas 模組文檔。
11. 依交付政策完成：提交並推送。完成驗證後建立聚焦 commit，確認目標 remote/branch 後推送；若 remote 或 branch 不清楚，推送前必須先詢問。

## 編輯前摘要

當使用者或專案規則期待確認時，先摘要並等待確認：

- 原始要求
- 主模組與邊界模組
- 必須維持不變的行為
- 重構邊界與非目標
- 重構前後行為，包含使用者可見行為與內部責任切分
- 預計修改檔案
- 要執行的測試或檢查

## 交付政策

提交並推送。完成驗證後建立聚焦 commit，確認目標 remote/branch 後推送；若 remote 或 branch 不清楚，推送前必須先詢問。
